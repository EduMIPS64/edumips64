## Summary

Fixed a build-breaking Windows CI failure caused by illegal colons in timestamped filenames. Scribe runs had written `.squad/log/` and `.squad/orchestration-log/` files with ISO-8601 timestamps containing colons (e.g., `2026-06-08T13:55:00Z-tank.md`), causing `actions/checkout` to fail with exit code 128 on Windows CI → master Release workflow red.

**Root Cause:** Windows filesystem forbids colons in filenames; GitHub Actions' checkout step on Windows detects and rejects them.

**Fix:**
1. Renamed all 6 offending files to colon-free form (`2026-06-08T13-55-00Z-...`)
2. Added mandatory Windows-safe filename convention to Scribe charter
3. Activated windows-compatibility skill
4. Updated orchestration-log template

**Verification:** Release run 27144326413 on ed03753b — SUCCESS; both Windows CI jobs passed checkout (previous failure point); CodeQL + Code Quality green.

**Convention Change (now mandatory):** All timestamped files in `.squad/log/` and `.squad/orchestration-log/` use format `YYYY-MM-DDTHH-MM-SSZ` (hyphens, not colons). Example: `2026-06-08T14-35-00Z-task.md` instead of `2026-06-08T14:35:00Z-task.md`.
